# Enchainement de demo

Objectif : montrer un parcours produit coherent en peu de temps.

## Parcours propose

1. Connexion a l'application.
2. Ouverture de la liste des tables.
3. Acces a une table existante.
4. Passage par l'administration MJ :
   - membres ;
   - roles ;
   - invitation ciblee ou lien de groupe.
5. Consultation ou creation de la prochaine session.
6. Reponse RSVP cote membre si un compte de test est disponible.
7. Demarrage de la session live par le MJ.
8. Dans le live :
   - presence ;
   - chat ;
   - notes ;
   - jet de des ;
   - message prive ;
   - drawer d'outils.
9. Ouverture du centre de notifications.
10. Retour sur l'etat de la session ou fin/annulation selon le scenario.

## Points a preparer

- Avoir au moins une table de demo.
- Avoir un compte MJ et si possible un compte joueur.
- Preparer une session planifiee.
- Verifier les variables email seulement si l'envoi doit etre montre.
